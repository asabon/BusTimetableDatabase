# /push ワークフロー

このワークフローは、コミット済みの変更をリモートリポジトリへ反映させる手順を定義します。

## 前提条件
- 必要なコミットが完了していること。

## 手順

### 1. ブランチの確認
現在のブランチが正しい（作業用ブランチである）ことを確認する。

### 2. プッシュの実行
現在のブランチをリモート（origin）へプッシュする。

```bash
git push origin <BRANCH_NAME>
```

## エージェント（Antigravity）への注意事項
- プッシュ前に `git status` で未コミットの重要な変更が残っていないか軽く確認すること。
