# /create-pr ワークフロー

このワークフローは、作業が完了しブランチをプッシュした後に、Pull Request を作成する手順を定義します。

## 前提条件
- 修正内容がリモートリポジトリへプッシュされていること。
- PR のマージ先（通常は `main`）が正しいこと。

## 手順

### 1. PR 本文を作成する
文字化けやエスケープシーケンスの問題を防ぐため、必ず一時ファイルを使用して本文を作成する。

#### 本文に含めるべき内容:
- **概要**: 変更の目的と概要。
- **関連 Issue**: `Fixes #ID` の形式で関連する Issue をリンクする。
- **変更内容**: 具体的な修正箇所や実装のポイント。
- **検証結果**: 実行したテストや動作確認の結果。

### 2. 一時ファイルへの保存
本文を `pr_body.md` などの一時ファイルに保存する。

### 3. PR の作成
`gh` CLI を使用し、既存の Issue と紐づけて PR を作成する。

```bash
gh pr create --title "タイトル" --body-file pr_body.md --base main --head <BRANCH_NAME>
```

### 4. 後処理
- 作成された PR の URL をユーザーに報告する。
- 作成した一時ファイル (`pr_body.md`) を削除する。
- **重要**: AI は PR のマージを行わない。マージはユーザーに依頼すること。

## 注意事項
- PR のタイトルにも、必要に応じて Issue ID を含めると分かりやすい。
- レビューコメントがついた場合は、適宜修正を行い、再度プッシュすること。
