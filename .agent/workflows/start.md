# /start ワークフロー

このワークフローは、実装フェーズに移行する直前の「準備状況チェック」を行い、抜け漏れがないことを確認して作業を開始するための手順を定義します。

## 手順

### 1. エージェント指示書の読み込み
`.agent/AGENT.md` を読み込み、最新の基本ルールを把握する。

### 2. 準備状況のセルフチェック
エージェントは以下の項目を自動で確認し、不備があればユーザーに報告する。

- **Issue 連携**: 作業対象の GitHub Issue が特定されているか？ `docs/progress/task.md` に Issue ID が記載されているか？
- **作業ブランチ**: `main` 以外の適切な作業ブランチがチェックアウトされているか？
- **実装計画の有無**: `implementation_plan.md` が作成され、ユーザーの承認（または合意）が得られている状況か？
- **進捗管理ファイルの更新**: `docs/progress/task.md` が現在の状況に合わせて更新されているか？

### 3. ステータス報告と着手宣言
チェック結果をユーザーに簡潔に伝える。

- **問題がない場合**: 「準備が整いました。実装計画に基づき、[Issue ID] の作業を開始します」と宣言して作業に着手する。
- **不備がある場合**: 「[不備内容（例：ブランチが main のままです）] を確認しました。修正してから開始してよろしいでしょうか？」とユーザーに確認する。

## 注意事項
- このワークフローは、ユーザーから「作業を開始してください」や「/start」という指示があった際に呼び出されることを想定している。
- 単なる「再開 (/resume)」ではなく、「実行フェーズへのキックオフ」としての正確性を重視すること。
